KPI Generation Project

I led and implemented a solution to optimize the generation of KPIs for geolocation accuracy using dynamic SQL queries in Apache Superset.

To do this, I had to first analyze the existing static Python script to figure out its design flaws and potential ways to improve its accuracy and efficiency.

Second, after deciding that running SQL queries would expedite the process, I began developing these queries to be able to extract the most relevant statistics from the large geolocation database in an efficient way.

Once I had figured out the implementation of these queries, I evaluated Apache Superset to ensure charts, tables, and heat maps could be utilized efficiently.

Lastly, I led department-wide tutorial sessions during the rollout of the functionality.

Here is the table that shows the progression of time efficiency to generate KPIs for the 3 SOIs (Signals Of Interest) that I was tasked with:

Design Total Generation Time for VHF Total Generation Time for LBAND Total Generation Time for XBAND
Initial Design 1hr 45m 45m
Two queries: one to join desired columns and one to generate statistics (WITHOUT optimization) 45m 30m 15m
Two queries: one to join desired columns and one to generate statistics (WITH optimization) 15m 10m 5m
